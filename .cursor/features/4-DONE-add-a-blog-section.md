## Contents

Add a blog section to my portfolio page. 

This should be at /blog. It should have the same boilerplate, such as the header and logo, as all the other pages. It should be organized in a logical manner in descending chronological order from most recent post downward. There should also be a right-hand menu with the most recent five posts that you can click on to go to that post. ## First post

I would like the first post to be about the creation of this portfolio site. It should use the git history and possible, past history with Claude agents in building the portfolio site. It can also consult my conversations with my personal agent. I'll provide those conversations when we start working on this feature. 

The first post should use the same punchy, assertive language that the rest of the portfolio site uses. It should highlight my achievements in building the site without being showy or braggy. It should illustrate the skills that I learned in building out this portfolio site, namely:
- multi-agent orchestration
- careful planning of site structure and content
- pivoting when things didn't go my way 

A prime example of the latter is when my planned use of two competitive developer agents to build out the site itself didn't go according to plan. My idea was to spin up two developer agents that couldn't talk to each other and have them each build out their own version of the site, knowing that they were competing with another agent but not being able to interact with that agent in any way. My idea is that this would increase the quality of the final product. However, I got bogged down in prompt permission hell doing this approach. It turned out that the better approach was to just appoint one developer agent to do the site and then go through the planned QA and code review processes. I was able to salvage some of what one of the competitive agents had done, however. 

The post should outline the process and steps involved in building the portfolio site.
